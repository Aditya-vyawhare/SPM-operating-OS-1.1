# Skill: Booking Funnel Optimization

Use this skill to improve search-to-book conversion in corporate travel flows.

## Funnel map (default)

1. session start
2. search submitted
3. itinerary viewed
4. policy-compliant option selected
5. traveler details completed
6. payment success
7. booking confirmed

## Workflow

## 1) Quantify stage drop-offs

- Measure conversion between each stage
- Segment by mode, device, company, and route type

## 2) Identify root causes

Check for:

- fare refresh failures at checkout
- policy confusion or approval delays
- slow page/API response times
- payment failure reasons

## 3) Prioritize fixes

Use impact x effort scoring:

- high drop-off x low effort first
- high drop-off x high effort as roadmap item

## 4) Design interventions

- Fare lock or clearer fare-change alerts
- Policy pre-check before checkout
- Streamlined traveler profile autofill
- Payment retries with fallback options

## 5) Validate impact

- Run controlled experiments where possible
- Track conversion lift and guardrails (AOV, cancellation rate, support contacts)

## Standard deliverable

- Funnel diagnosis table
- Top 5 fixes with estimated GMV uplift
- Experiment and rollout plan
